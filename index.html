HTML Semantics: coding the content of a web page to give value and structure separate from the style or appearance of that content. Semantic code aids web accessibility and can improve search engine optimization. It also allows for easier editing or adding of page content since the design is created separately in the cascading style sheet.

HTML Block-level vs. Inline-level Elements: Block-level elements begin on a new line, are stacked upon one another, occupy any available width, and may be nested inside one another and may wrap inline-level elements. Inline-level elements are maintained within the flow of the document structure, are lined up one after the other, will occupy only the width of their content, and may be nested inside one another, but cannot wrap block-level elements. Block-level elements are used for larger pieces of content, such as paragraphs; whereas inline-level elements are used with smaller pieces of content, such as a few words.

Anchor Element: The anchor element <a> breaks the standard inline-level element rule by specifically being able to wrap either block- or inline-, or any other level elements in order to enable entire blocks of content on a page to become links.

Relative & Absolute Paths: The relative path is a link that references a page on the same website and does not include the domain (.com, .org, .edu, etc.) in the href attribute value; it includes only the filename of the page being linked to. If the page is located within a different directory or folder, the href attribute value needs to reflect this also. An example of this would be either the <a href="about.html">About</a> or <a href="pages/about.html">About</a> The absolute path is a link that references a page on a separate or another website and must include the entire url in the href attribute value, starting with the http and ending with the domain (.com, .org, .edu, etc.). An example would be <a href="https://www.google.com/">Google</a>

Structural Elements are HTML5 semantic elements and tags intended to give meaning to the organization of a web page. They are all block-level elements and do not have any implied position or style and can be used multiple times per page.

Understanding Hyperlinks

HTML Links connects a webpage to another URL address. The syntax uses the HTML <a> tag with the attribute href (hyperlink reference), which indicates the link's destination. The link text is visible to the reader and clicking on the link sends the reader to the specified URL address.

To break down the basics: the <a> element is an anchor to define a hyperlink. the href attribute references the URL address to link to. If there isn't a href attribute, it is only a placeholder for a hyperlink. the URL address is necessary to either be absolute (links to an outside page) or relative (links to a local page). The link text is the displayed text on the webpage and what is to be clicked on.

An example of a basic link: W3Schools.com HTML Links

By default, the linked page will be displayed in the current browser window. To change this default, use the target attribute to specify where to open the document. The target attribute can have one of the following values:

  • _self -  Default. Opens the link in the same window/tab being used.
  • _blank -  Opens the link in a new window or tab.
  • _parent - Opens the link in the parent frame.
  • _top -  Opens the link in the full body of the window.
An example of using target="_blank" to open in a new window: W3Schools HTML Links

Link to Email: To create an email link, the href attribute value needs to start with mailto: followed by the email address to which the email should be sent. For example, <a href="mailto:thomasdown@strict.org">Email Me</a>. To include a subject line, add the subject= parameter after the email address. Use a question mark ? between the email and subject parameter to bind the two; to add body text, use the body= parameter in the href attribute value. To bind the two parameters yet keep them separate, the ampersand & is required. If using multiple words within a subject line or body text, the spaces must be encoded using %20; line breaks must be encoded using %0A or <br> Example: <a href="mailto:thomasdown@strict.org?subject=More%20Info& body=Please%20tell%20me%20more">Email Me</a>

Open Links in New Window: To create a link that opens into a new browser window, use the target attribute with a value of _blank. The target attribute determines exactly where the link will be displayed, and the _blank value specifies a new window. The code example is <a href="http://google.com/" target="_blank">Google</a>

Same Page Links: To create a same-page link, such as "Back to top" links that return a user to the top of a page, create an id attribute on the element you wish to link to, then using the value of that id attribute within an anchor element's href attribute. For example, to code a "back to top" link - place the id attribute of top on the <body> element, then add #top (pound sign with value name) to link the beginning of the <body> element. <body id="top"> <a href="#top">Back to top</a>

Unordered List
A listing of related terms whose order does not matter. To create an unordered list, use the <ul> element with each list item marked symbolically with the <li> element.
Ordered Lists
A listing of terms whose presentation of order is important. To create an ordered list, use the <ol> element with each item marked numerically with the <li> element.

Ordered lists have two attributes available for use - start and reversed. Start attribute defines the number from which an ordered list should start; by default, the list starts at 1. When using the start attribute on the <ol> element, you can identify exactly what number the ordered list will begin counting from; it uses only integer values. For example: <ol start="20">
The reverse attribute allows a list to appear in reverse order. It is a Boolean attribute- it accepts only true or false, with False as the default value. When the attribute reversed appears in the <ol> element, the value becomes True. For example: <ol reversed>


Understanding Images

Adding Images: Images can be added to a page using the <img> inline element. It is self-containing, so it doesn't wrap any other content and exists as a single tag. For the <img> element to work, a src attribute and value must be included to specify the source of the image; typically the src attribute is a URL. In addition to the src attribute is the alt or alternate text attribute to describe the image for web accessibility and SEO purposes. Supported online image file formats include jpg, gif, and png; with jpg and png being the most widely used.
Code format example: <img src=https://www.ferialyouakim.com/wp-content/uploads/2017/03/AdobeStock_85718241.jpg" alt="sunflower in field with blue sky background">

When adding images, it is helpful to size the image to fit appropriately on the page. One way is to resize the actual file using an editor; this can be done only for local files, not web images. The second, more effective way, is to use the width and height attributes for the image tag. When using only the width attribute in pixels, the browser will adjust the height so it stays proportional to the width. When opting to create a specific size for the image to fit, use both the width and height attribute. Lastly, using a percentage of the image size or container will resize to that percentage- for example, width=50% reduces the image to 50% of what it should be, not the original picture size.

Images are not technically inserted into a web page, they are linked to web pages. The <img> element creates a container for the referenced image. Alone it is empty, it must contain attributes - specifically the src and alt. The src attribute specifies the path to the image and the alt specifies an alternate text for the image. The URL path is requested by the browser at the time the web page loads and inserts the image. If the browser cannot locate the image source, the broken link icon and the alt text are shown. Another attribute that's helpful is the either the style attribute to specify the width and/or height of an image, or the individual width and height attributes. The style attribute requires the format use of either pixels or percentage; the width/height attribute is pixels. It is advised to use the style attribute to prevent the styles sheets from potentially changing it. If the image file is large, it may take time to load. A good practice is to set the style width and height to prevent the web page from flickering while the image loads.

Image floating allows an image to float either to the right or to the left of a text. This is added to the style attribute. An example of the syntax: To float right of the text: <img src="https://cdn.britannica.com/57/115657-004-1C7267FE/St-Augustine-Canterbury-Abbey-England-Kent.jpg" alt="Abby in Kent, England" style="float:right; width:80px;height:70px;"> To float left of the text: <img src="https://cdn.britannica.com/57/115657-004-1C7267FE/St-Augustine-Canterbury-Abbey-England-Kent.jpg" alt="Abby in Kent, England" style="float:left; width:80px;height:70px;">

Creating an Image Map

An Image Map creates clickable areas on an image. It is defined by the <map> tag. To begine the creation of a clickable image, you use the <img> tag with the attribute usemap, whose value starts with a hash tag # followed by the name of the image map; this is used to create a relationship between the image and the image map. An example is <img scr=https://o.quizlet.com/NNq5-Elv2O1XuSir4tADZg_b.png" alt="Parts of a computer" usemap="#computerparts">

To create the image map, use the <map> element with the name attribute equal to the created usemap attribute in the <img> tag. Then add the clickable areas using the <area> element to define the shape and coordinates of the area available to be clickable. The Shape of the area must be defined using one of the following values:

  • rect - defines a rectangular region
  • circle - defines a circular region
  • poly - defines a polygonal region
  • default - defines the entire region

The coordinates for each shape come in pairs, one for the x-axis and one for the y-axis. Each shape has a formula for determining these coordinates.

  • shape="rect" coordinate starts at the top left corner X,Y of the area and then the bottom right corner X,Y so that (34,44,270,350) equals 34px from left margin,44px from top,270px from left margin,350px from top.
  • shape="circle" coordinate starts at the center of the circle area X,Y and then specifies the radius X of the circle so that (337,300,44) equals 337px from left margin,300px from top is the center and 44px is the radius.
  • shape="poly" coordinates has several X,Y points which custom creates a shape formed with straight lines that conforms to any image. The multiple coordinate pairs are determined by outlining the shape using the left margin (X) and top margin (Y) for all edges. a poly coordinate example is (140,121,103,161,40,259,37,322,85,355,191,270,204,160,181,116) which desinates an oblong shape of a croissant.

Understanding Audio

Adding Audio: Audio files can be added to a page using the <audio> inline element. Like the <img> element, the <audio> element uses the src attribute with a source URL; however it requires an opening and closing tag. Basic example: <audio src="https://youtu.be/_LVfsrS7fUk"></audio> There are several Audio attributes available, the first 3 are Boolean attributes set to True when present. The following below are the most commonly used attributes:

autoplay
A Boolean attribute that will automatically play an audio file upon loading. <audio src="https://youtu.be/_LVfsrS7fUk" autoplay><audio>
controls
A Boolean attribute that will display the audio controls on a page, including play and pause, seek, and volume controls. <audio src="https://youtu.be/_LVfsrS7fUk" controls></audio>
loop
A Boolean attribute that will cause an audio file to repeat continually, from beginning to end. <audio src="https://youtu.be/_LVfsrS7fUk" loop></audio>
preload
An attribute that helps identify what, if any, information about the audio file should be loaded before the clip is played. It accepts three values: none, auto, and metadata. The none value won't preload any information about the file, while the auto value will preload all information about the file. The metadata value will preload any available metadata information, but not all the information. When the preload attribute isn't present on the <audio> element, all information is loaded as if set to auto; therefore a good practice is to use the metadata or none when the audio file isn't essential to a page.

Understanding Tables

Creating Tables: HTML tables provide a straightforward way to mark up structured tabular data and to display that data in a form that is easy to read and digest. The table data is contained within columns and rows; at minimum a table consists of the <table>, <tr> (table row), and <td> (table data) elements. For greater structure and additional semantic value, tables may include the <th> (table header) element.

Structured elements of a table
<table> Table element initializes a table on a page
<tr> Table Row element initiates the row on a page; numberous rows can be added
<td> Table Data element displays the data included in the row; each <td> element listed one after the other will create columns within a table row
</td> required closing tag added at the end of each data element
</tr> required closing tag added to close the Table Row element
</table> required closing tag added to close the Table element

Table Header is used to designate a heading for a column or rows of cells. The <th> element creates a cell for data and adds semantic value to the content. Table headers may be used within both columns and rows; the scope attribute helps identify exactly what content a table header relates to and is indicated with the values col (all cells within the column) row, (all cells within the row), colgroup, and rowgroup. The first two are most commonly used, an example is <th scope="col">Item</th> <th scope="row">Utilities</th>

Table Caption: A caption helps users identify what the table pertains to and what data to expect within it. The <caption> element must come immediately after the opening <table> tag, and it is positioned at the top of a table by default.

Table Head, Body & Foot: The content within tables can be broken up into multiple groups, including a head, a body, and a foot. The <thead> (table head) wraps the heading row or rows of a table to denote the head; it's placed at the top of a table, after any <caption> element and before any <tbody> element. The <tbody> element contains the primary data within a table; it's place after the <thead> element. The <tfoot> element contains data that outlines the contents of a table; it's placed after the <thead> element, but either before or after the <tbody> element.

Combining Multiple Cells: When two or more cells need to be merged into one without breaking the overall row or column layout, use the colspan and rowspan attributes. These two attributes work on either the <td> or <th> elements. The colspan attribute merges a single cell across multiple columns within a table; while the rowspan attribute is used to merge a single cell across multiple rows. Each attribute accepts an integer value that indicates the number of cells to merge, with 1 being the default value. Examples include: <th scope="col" colspan="3">Item description</th> or <td colspan="4">Cost</td>
